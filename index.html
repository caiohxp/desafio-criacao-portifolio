HTML

HTML é uma linguagem de marcação utilizada na construção de páginas na Web. Documentos HTML podem ser interpretados por navegadores. A tecnologia é fruto da junção entre os padrões HyTime e SGML.

HTML5 é a quinta versão da linguagem HTML. Esta nova versão traz consigo importantes mudanças quanto ao papel do HTML no mundo da Web, através de novas funcionalidades como semântica e acessibilidade. Possibilita o uso de novos recursos antes possíveis apenas com a aplicação de outras tecnologias. Sua essência tem sido melhorar a linguagem com o suporte para as mais recentes multimídias, enquanto a mantém facilmente legível por seres humanos e consistentemente compreendida por computadores e outros dispositivos (navegadores, parsers etc).

Acesse o site oficial:

https://www.w3.org/html/
Prós
  • Fácil de aprender e usar.
  • HTML é leve e rápido de carregar.
  • HTML pode ser compatível com todos os navegadores, se não for compatível com todos os navegadores.
Contras
  • Ele não pode produzir saída dinâmica sozinho, pois é uma linguagem estática.
  • Ele pode criar apenas páginas estáticas e simples, portanto, se quisermos páginas dinâmicas, o HTML não será útil.
  • Necessário para escrever uma grande quantidade de código apenas para criar uma página da web simples.

CSS

CSS é chamado de linguagem Cascading Style Sheet e é usado para estilizar elementos escritos em uma linguagem de marcação como HTML. O CSS separa o conteúdo da representação visual do site. Pense na decoração da sua página. Utilizando o CSS é possível alterar a cor do texto e do fundo, fonte e espaçamento entre parágrafos. Também pode criar tabelas, usar variações de layouts, ajustar imagens para suas respectivas telas e assim por diante.

CSS instrui a exibição do HTML sobre como o site será exibido no final do usuário. Vamos dar uma olhada nas vantagens e desvantagens do CSS.

Acesse o site oficial:

https://www.w3.org/Style/CSS/Overview.en.html
Prós
  • CSS desempenha um papel importante, ao usar CSS você simplesmente precisa especificar um estilo repetido para o elemento uma vez e usá-lo várias vezes, pois o CSS aplicará automaticamente os estilos necessários.
  • Ajuda a formar mudanças espontâneas e consistentes.
  • Ele tem o poder de reposicionamento. Isso nos ajuda a determinar as mudanças na posição dos elementos da web que estão na página.
Contras
  • Com CSS, o que funciona com um navegador nem sempre funciona com outro. Os desenvolvedores da web precisam testar a compatibilidade, executando o programa em vários navegadores.
  • CSS funciona de maneira diferente em navegadores diferentes. O IE e o Opera suportam CSS como lógicas diferentes.
  • Existem vários níveis que criam confusão para não desenvolvedores e iniciantes.

Exemplo da Sintaxe do CSS:

body
{

background: black;
color: white;
font-size: 14px;

}

Sass

Sass é uma linguagem de script que é interpretada ou compilada em Cascading Style Sheets (CSS). SassScript é a linguagem em si. Sass consiste em duas sintaxes. A sintaxe original, chamada de "sintaxe indentada", usa uma sintaxe semelhante a Haml.[4] Ela usa indentação para separar blocos de código e caracteres de nova linha para separar regras. A sintaxe mais recente, "SCSS", usa formatação de bloco, como a de CSS. Esta usa chaves para designar blocos de código e ponto-e-vírgula para separar linhas dentro de um bloco. Os arquivos com sintaxe de indentação e SCSS são tradicionalmente dados as extensões .sass e .scss, respectivamente.

    JavaScript

    JavaScript é uma linguagem de script orientada a objetos, multiplataforma. É uma linguagem pequena e leve. Dentro de um ambiente de host (por exemplo, um navegador web) o JavaScript pode ser ligado aos objetos deste ambiente para prover um controle programático sobre eles.

    JavaScript tem uma biblioteca padrão de objetos, como: Array, Date, e Math, e um conjunto de elementos que formam o núcleo da linguagem, tais como: operadores, estruturas de controle e declarações. O núcleo do JavaScript pode ser estendido para uma variedade de propósitos, complementando assim a linguagem:

    • O lado cliente do JavaScript estende-se do núcleo linguagem, fornecendo objetos para controlar um navegador web e seu Document Object Model (DOM). Por exemplo, as extensões do lado do cliente permitem que uma aplicação coloque elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e de navegação da página.
    • O lado do servidor do JavaScript estende-se do núcleo da linguagem, fornecendo objetos relevantes à execução do JavaScript em um servidor. Por exemplo, as extensões do lado do servidor permitem que uma aplicação comunique-se com um banco de dados, garantindo a continuidade de informações de uma chamada para a outra da aplicação, ou executar manipulações de arquivos em um servidor.

    Acesse o site oficial:

    https://www.javascript.com
    Prós
    • Independentemente de onde você hospeda o JavaScript, ele sempre é executado no ambiente do cliente para economizar muita largura de banda e tornar o processo de execução mais rápido.
    • A maior vantagem do JavaScript é a capacidade de suportar todos os navegadores modernos e produzir um resultado equivalente.
    • JavaScript funciona bem com outras linguagens e pode ser utilizado em uma grande variedade de aplicativos.
    Contras
    • O principal problema ou desvantagem em JavaScript é que o código está sempre visível para qualquer pessoa que possa visualizar o código JavaScript.
    • JavaScript geralmente é interpretado de maneira diferente por navegadores diferentes. Isso torna um tanto complexo ler e escrever código para vários navegadores
    • Não importa a proporção que o JavaScript interpreta rapidamente, o JavaScript DOM (Document Object Model) é lento e pode ser uma renderização nunca rápida com HTML.

    Exemplo de Código em JavaScript

    function soma(a, b) {
    return a + b;
}

    const resultado = soma(a, b);
    console.log(resultado); // Output: 25

    TypeScript

    TypeScript é uma linguagem de programação de código aberto desenvolvida pela Microsoft. É uma linguagem que adiciona recursos de tipagem estática ao JavaScript, permitindo a criação de código mais robusto e escalável. O TypeScript é compilado para JavaScript puro e é compatível com qualquer navegador, sistema operacional ou ambiente que suporte JavaScript.

    A principal característica do TypeScript é a adição de tipos estáticos ao JavaScript. Isso significa que você pode declarar o tipo das variáveis, parâmetros de função, retorno de função e propriedades de objeto. Esses tipos ajudam a detectar erros em tempo de compilação e fornecem um melhor suporte a IDEs, permitindo a autocompleção de código e a detecção de erros durante o desenvolvimento.

    • O lado cliente do JavaScript estende-se do núcleo linguagem, fornecendo objetos para controlar um navegador web e seu Document Object Model (DOM). Por exemplo, as extensões do lado do cliente permitem que uma aplicação coloque elementos em um formulário HTML e responda a eventos do usuário, como cliques do mouse, entrada de formulário e de navegação da página.
    • O lado do servidor do JavaScript estende-se do núcleo da linguagem, fornecendo objetos relevantes à execução do JavaScript em um servidor. Por exemplo, as extensões do lado do servidor permitem que uma aplicação comunique-se com um banco de dados, garantindo a continuidade de informações de uma chamada para a outra da aplicação, ou executar manipulações de arquivos em um servidor.

    Acesse o site oficial:

    typescriptlang.org
    Prós
    • Tipagem Estática: O TypeScript adiciona um sistema de tipos estáticos ao JavaScript, permitindo a detecção de erros em tempo de compilação. Isso ajuda a encontrar erros comuns, como passar argumentos incorretos para uma função ou acessar propriedades indefinidas.
    • Maior Escalabilidade: Com a adição de tipos estáticos, o TypeScript torna o código mais robusto e escalável. À medida que o código-base cresce, é mais fácil fazer alterações e refatorações sem introduzir erros. Além disso, a tipagem estática permite melhor suporte a ferramentas como IDEs, que podem fornecer recursos como autocompletar e navegação no código.
    • Maior Suporte a Ecossistema: O TypeScript é amplamente adotado na comunidade de desenvolvimento. Muitas bibliotecas e frameworks populares, como Angular, React e Vue.js, têm suporte nativo ao TypeScript. Além disso, o TypeScript é suportado por várias ferramentas de desenvolvimento, como editores de código, IDEs e ferramentas de teste.
    Contras
    • Curva de Aprendizado: O TypeScript adiciona uma camada adicional de complexidade ao JavaScript, especialmente para desenvolvedores que estão familiarizados apenas com JavaScript. É necessário aprender os conceitos de tipos, anotações de tipo e como trabalhar com as peculiaridades do sistema de tipos do TypeScript.
    • Compilação Necessária: O TypeScript requer um processo de compilação para converter o código TypeScript em JavaScript. Isso significa que, antes de poder executar o código, é necessário executar o compilador do TypeScript. Embora existam ferramentas e configurações que podem facilitar essa tarefa, a compilação pode adicionar um passo adicional ao fluxo de desenvolvimento.
    • Sobrecarga Adicional: A adição de tipos estáticos pode aumentar a quantidade de código necessário para descrever os tipos corretamente. Isso pode resultar em um código-fonte mais longo e em uma curva de produtividade inicialmente mais lenta. Além disso, pode ser necessário lidar com situações em que a inferência de tipo automática do TypeScript não é suficiente e é necessário anotar explicitamente os tipos.

    Exemplo de código em TypeScript

    function soma(a: number, b: number): number {
    return a + b;
    }
    const resultado: number = soma(2, 3);
    console.log(resultado); // Output: 5

    PHP

    O nome PHP significa pré-processador de hipertexto e denota uma linguagem de script do lado do servidor, que sugere que os aplicativos escritos nele são executados em servidores da web e não dependem do navegador online. A sintaxe da linguagem PHP é análoga à linguagem C. Foi criado por Rasmus Lerdorf e surgiu em 1995. PHP está sendo amplamente utilizado no desenvolvimento de aplicações web e se tornou uma das principais linguagens para desenvolvedores fazerem novas aplicações.

    Os principais sites de redes sociais como o Facebook e organizações de renome como a Universidade de Harvard são compatíveis com PHP, o que torna o PHP popular e aumenta sua credibilidade.

    No entanto, com o passar dos anos, sua área de uso mudou, e hoje em dia a linguagem de codificação PHP é classificada entre as ferramentas de programação mais simples e mais quentes para desenvolvimento web, graças às suas muitas virtudes que podem ser o alvo principal deste texto. é considerada uma tecnologia realmente eficaz que oferece um processo de desenvolvimento conveniente com muitas ferramentas adicionais para auxiliá-lo. Na verdade, de acordo com o reconhecimento do índice de linguagem de programação (PYPL), o PHP é a quinta linguagem de codificação mais quente do mundo.

    Acesse o site oficial:

    https://www.php.net
    Prós
    • A vantagem mais importante do PHP é que ele é open source e não tem custos. Freqüentemente, são baixados em qualquer lugar e prontamente disponíveis para uso em eventos de aplicativos da web.
    • O aplicativo pode ser facilmente carregado, baseado em PHP e conectado ao banco de dados. é usado principalmente devido à sua taxa de carregamento mais rápida na Internet lenta e velocidade do que outra linguagem de programação.
    • Possui suporte de biblioteca poderoso para usar vários módulos de função para representação de dados.
    Contras
    • Não é tão seguro devido ao seu código aberto, porque os arquivos de texto ASCII geralmente estão facilmente disponíveis.
    • Não é adequado para aplicativos gigantes da web baseados em conteúdo.
    • Possui um tipo fraco, o que pode causar dados e conhecimento incorretos ao usuário.

É amplamente utilizado na indústria de jogos e possui uma grande comunidade de desenvolvedores e recursos disponíveis.

    unity.com/pt
